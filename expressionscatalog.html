Catalog of Commonly Used Funky Trees Expressions


This page provides a list of useful general FT expressions.



Warning:

This section is simply meant to provide a catalog of premade, generalized, and conventionally accepted FT expressions. This guide will not actually teach you on the details of how to use these FT inputs. Check the FT Guide page for a tutorial on Funky Trees.

Common Expressions List

This list is will be categorized as the number of general FT expressions increase. Currently it sits as a simple table list. In the table, x indicates some variable that you can manipulate and set.


Simple Activation Formulas (General Purpose)

Purpose General Form Explanation
Activates a part when Altitude is higher than some value Math-safe form: clamp01(Altitude > x) / Simplified Boolean Form: Altitude > x Math-safe form: will return 1 when Altitude is larger than x, 0 otherwise.
Boolean expression: will return true when Altitude is larger than x, false otherwise.
Activates a part when Altitude is lower than some value Math-safe form: clamp01(Altitude < x) / Simplified Boolean Form: Altitude < x Math-safe form: will return 1 when Altitude is lower than x, 0 otherwise.
Boolean expression: will return true when Altitude is lower than x, false otherwise.
Activates a part when speed is higher than some value Math-safe form: clamp01(TAS > x) / Simplified Boolean Form: TAS > x Math-safe form: will return 1 when TAS, or speed, in meters per second is lower than x, 0 otherwise.
Boolean expression: will return true when TAS (speed in m/s) is larger than x, false otherwise.
Activates a part when speed is slower than some value Math-safe form: clamp01(TAS < x) / Simplified Boolean Form: TAS < x Math-safe form: will return 1 when TAS, or speed, in meters per second is lower than x, 0 otherwise.
Boolean expression: will return true when TAS (speed in m/s) is slower than x, false otherwise.

Specific-Purpose Formulas (Utility)

Purpose General Form Explanation
Linearly decreasing wheel rotation Ctrl * clamp01(-a*GS + b) Ctrl is any control input you want to use for wheel rotation, for example Yaw. a is the linear slope parameter, determining the rate at which you lose control over the wheels. b is the offset parameter, used to change the point at which wheel rotation angle begins to get reduced.
Weapon Reload Timer (1 - smooth(clamp01(-rate(ammo("x"))), clamp01(-rate(ammo("x")))*pow(10, 4) + y)) This script returns 0 when the weapon x is fired, and linearly progresses to the value 1 at a fixed limit of y amount per second. That is, for the entire progression to last x seconds, the y should be 1/x. The frontal segment of "1 - " can be removed to have a 1 to 0 progression, but this is arguably less useful.
Change Count Observer sum(abs(rate(x))) This script returns the number of times a change in the variable x has occurred (best fit for boolean or stepped-change variables). For example, ammo("Cannon") can be substituted for x to count the number of shots fired (with some adjustment), or the variable LandingGear to count the number of times the LandingGear button was pressed.
